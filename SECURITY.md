# Security Policy

ScieMD is local-first software for scientific Markdown documents. Please report security
issues privately before opening a public issue.

## Supported Versions

Security fixes are handled on the current development branch until formal release branches
exist.

## Reporting a Vulnerability

Report vulnerabilities to `info@scienfy.com` or through GitHub private security
advisories. Include:

- Affected version or commit.
- Reproduction steps.
- Impact and affected operating system.
- Whether document data, local files, exports, or extension webviews are involved.

Do not include private research data in reports unless it is necessary and you have
permission to share it.
